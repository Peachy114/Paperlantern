Trending system

Do not rank trending using only total views. Older works will almost always win.

Use recent activity from a table such as:

work_activity_events
- id
- work_id
- user_id
- session_id
- event_type
- created_at

Possible event_type values:

view
chapter_read
like
rating
bookmark
follow
comment
review
share
Suggested trending score
trending_score =
unique_readers_24h × 4
+ chapter_reads_24h × 2
+ likes_24h × 5
+ follows_24h × 6
+ bookmarks_24h × 4
+ comments_24h × 3
+ shares_24h × 8
Trending type	Filter	Purpose
Trending Today	Activity from the last 24 hours	Shows what is currently active
Trending This Week	Activity from the current seven days	Shows weekly momentum
Trending This Month	Activity from the current month	Shows longer-term momentum
Trending Manga	works.type = manga	Manga-only trending
Trending Manhwa	works.type = manhwa	Manhwa-only trending
Trending Manhua	works.type = manhua	Manhua-only trending
Trending Novels	works.type = novel	Novel-only trending

Status widgets

Use a status field inside works.

ongoing
completed
hiatus
cancelled
dropped
coming_soon
Widget	Query	Purpose
Ongoing	status = ongoing	Shows currently publishing works
Completed	status = completed	Shows finished works
Hiatus	status = hiatus	Shows temporarily paused works
Dropped	status = dropped	Shows works no longer translated or supported
Cancelled	status = cancelled	Shows officially cancelled works
Coming Soon	status = coming_soon	Shows upcoming works

You may also add:

publication_status
translation_status

For example:

publication_status = ongoing
translation_status = hiatus

This is useful because the original work may still be ongoing while your translation is paused.

The critical field is:

type

Values:

manga
manhwa
manhua
novel

Then any widget can support everything:

Trending All Works
Trending Manga
Trending Manhwa
Trending Manhua
Trending Novels

The only difference is the query filter:

$query->where('type', 'novel');

or:

$query->whereIn('type', ['manga', 'manhwa', 'manhua']);
